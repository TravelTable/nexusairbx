# Acceptance Tests and Risk Register

**Audit date:** 2026-07-18  
**Scope:** shared rebuild contracts, Prompt 2 (Roblox asset platform), and Prompt 3 (reliable autonomous agent and Studio runtime)  
**Purpose:** define objective release gates. This document does not authorize or begin either rebuild.

## Evidence language

- **Confirmed** means the conclusion is directly supported by the cited repository implementation or by a named test run performed during this audit.
- **Strong inference** means multiple implementation facts point to the conclusion, but the audit did not reproduce it in a deployed environment.
- **Unknown** means the repository or the audit evidence is insufficient to decide.
- A **target requirement** is a future pass/fail condition, not a claim about the current system.

All percentages below are calculated over the named test corpus. “Zero” means no observed violation in that corpus. A criterion is not satisfied by a mocked happy path when its wording calls for a real emulator, process boundary, Studio plugin, or Roblox sandbox operation.

| P3-05 | A Studio disconnect pauses only Studio-dependent steps. | P3-07 | Every reversible mutation declares a rollback recipe and pre-change snapshot; rollback itself is verified and recorded as a new event. | Fault after each reversible mutation in the E2E corpus. | P3-08 | Stream updates are projections of the durable ledger and reconnect from a monotonic cursor. A user sees the same terminal state after reload as before it. | Disconnect/reconnect the stream after every event and replay with old, current, and invalid cursors. ## Test strategy

### Unit and contract tests

- Validate every shared schema at both producer and consumer boundaries: capability snapshots, task/step/event/checkpoint records, Studio command/result, typed error, manifest/file, asset/icon-pack/style-profile, Roblox relationship, and idempotency receipt.
- Use table-driven transition tests and property-based generation for task, command, OAuth, upload, moderation, and replacement state machines. Every invalid transition must fail without mutating state.
- Freeze retry-policy tests around error class, maximum three attempts, backoff/jitter bounds, deadline, and restart-persisted attempt count.
- Freeze manifest comparison tests around revision identity, content hash, added/changed/deleted paths, stale base, and simultaneous web/Studio edits.
- Test prompt assembly and context selection with explicit token budgets: identity and safety instructions must never be truncated; only authoritative context can satisfy a required-context predicate.
- Validate asset metadata, style-profile hashes, transparent-background checks, price-policy bounds, replacement lineage, moderation mapping, and visibility policy.

### Integration tests

- Run Firestore and Redis-backed tests with real transactions, at least two API/worker processes, and independent caches. In-memory fakes are useful for unit tests but cannot satisfy atomicity or multi-instance gates.
- Exercise OAuth authorization state, PKCE, nonce, callback replay, encrypted token persistence, refresh fencing, revocation, and capability snapshots against a deterministic Roblox adapter; reserve a smaller sandbox suite for live API compatibility.
- Drive model tool dispatch through the capability registry, task ledger, queue, worker, Studio transport adapter, checkpoint writer, verifier, and stream projection. Assert both persisted records and emitted events.
- Run asset generation, storage, upload, operation polling, moderation, ID persistence, search, Studio application, replacement, and rollback through the canonical asset aggregate.
- Run compatibility reads/writes against pre-backfill and post-backfill data until migration parity and rollback criteria pass.

### Fault-injection and recovery tests

- Place a deterministic fault point immediately before and after every external call and every ledger, checkpoint, lease, acknowledgement, projection, and verifier write.
- Kill browser, API, worker, and plugin independently; restart them in a different order; advance a fake clock across leases, token expiry, retry delay, and command expiry.
- Inject timeout-before-accept, accept-then-lost-response, malformed response, duplicate response, reordered event, stale lease holder, partial icon pack, moderation delay/rejection, manifest conflict, and rollback failure.
- Compare the rebuilt projection with the authoritative ledger after every fault. The required invariant is convergence without an untracked external effect.

### Security and isolation tests

- Build an authorization matrix for anonymous, authenticated non-owner, project member by role, tenant admin, service worker, and Studio session principal.
- Attempt horizontal and vertical access across asset catalog/search, generation, upload/poll, manifest, Studio command, task stream, checkpoints, shared-universe resources, and admin endpoints.
- Test callback replay, wrong OAuth state/nonce/verifier, token substitution, revoked credentials, stale encrypted-token writes, request-hash substitution under a reused idempotency key, and untrusted browser creator/universe metadata.
- Fuzz JSON/body/file limits and typed-error serialization; verify logs, streams, and responses never expose OAuth tokens, encryption material, private storage URLs, or another tenant’s identifiers.
- **Confirmed current concern:** Redis reconnect is disabled and client error events are ignored in `backend/src/lib/securityStore.js:168-197`; rate-limit/security-store behavior under Redis loss therefore needs an explicit fail-open/fail-closed policy test.
