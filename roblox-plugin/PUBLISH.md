# Publish Or Update The NexusRBX Studio Plugin On Roblox

The install artifact is **one bundled script** (~4,800 lines), not the `roblox-plugin/src/` folder.

If Roblox Output shows errors like:

- `Plugin.src.commands.readTools`
- `Plugin.src.Main.server`
- `cloud_XXXXX.Plugin.src...`

you published the **wrong folder shape**. Delete that plugin version and republish using the steps below.

## Build the publish artifact

From the repo root:

```sh
npm run plugin:install
```

This writes:

- `roblox-plugin/NexusRBXStudioBridge.plugin.lua` — paste source
- `roblox-plugin/build/NexusRBXStudioBridge.rbxmx` — single-script import/publish file

## First-time publish (Roblox cloud plugin)

1. Run `npm run plugin:install`.
2. Open Roblox Studio (empty baseplate is fine).
3. Create **one** new `Script` in `ServerStorage` (name it `NexusRBXStudioBridge`).
4. Open `roblox-plugin/NexusRBXStudioBridge.plugin.lua`, select all, paste into that script.
5. Confirm the script starts with:
   ```lua
   -- NexusRBX Studio Bridge
   -- Generated by roblox-plugin/build/bundle-plugin.js.
   ```
6. Confirm the script does **not** contain a `src/` folder in Explorer.
7. Right-click **that script only** → **Publish as Plugin** (or **Save as Plugin** then publish from Creator Dashboard).
8. Enable **Allow HTTP Requests** in any experience that uses the bridge.

## Update an existing published plugin (your `cloud_...` asset)

1. Run `npm run plugin:install` to get the latest bundle.
2. In Studio: **Plugins → Manage Plugins**.
3. **Disable or uninstall** the broken NexusRBX plugin (the one throwing `Plugin.src...` errors).
4. Open the plugin for editing:
   - **Plugins → Manage Plugins → your plugin → Edit**, or
   - Creator Dashboard → **Creations → Plugins → select asset → Edit**.
5. In the plugin place, **delete** any `Plugin` folder that contains `src/`, `Build/`, or multiple scripts.
6. Create **one** new `Script` and paste the full contents of `roblox-plugin/NexusRBXStudioBridge.plugin.lua`.
7. Publish the update to the **same plugin asset** (Update / Publish to Roblox).
8. Restart Studio, reinstall/enable the plugin, and **Pair Studio** again from `/ai`.

## Do not publish

- `roblox-plugin/src/` (dev sources — invalid alone)
- A `Plugin` folder with multiple scripts
- `ServerStorage.Plugin` copied from a place
- Anything whose Output path includes `.src.commands.`

## Local-only install (no Roblox cloud)

```sh
npm run plugin:install
```

Then restart Studio. File path (macOS):

`~/Documents/Roblox/Plugins/NexusRBXStudioBridge.rbxmx`
